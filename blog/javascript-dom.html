An analogy to describe JavaScript and its relationship to HTML and CSS?

You can describe JavaScript and it’s relationship to HTML and CSS like building a modern new house.

A new house requires a foundation,walls and structure(HTML). It also requires styling, design and looks (CSS). A new modern house also needs interactivity (JavaScript).

This analogy wouldn’t have been possible 10 years ago in New Zealand but being a tradie working in new homes. More often than not now, every new house I’ve worked on has some form of functionality. A good example of interactivity would be WiFi controlled components like apps for appliances like heating, sensors for lighting and security or even painted walls you can tap that conduct electricity to turn off lights.

Hyper Text Markup Language (HTML) defines the content and structure of a website. HTML holds everything together and provides a foundation and structure for all the information.

If HTML is the structure of a website, that makes CSS the styling and look of the web page. CSS styles the different content in HTML with different attributes.

Last but not least is JavaScript. Every new house needs interactivity as a selling point in 2022. It’s the same with web pages. JavaScript interactivity can take many forms.

Some examples are:

Opening and closing windows
Displaying dialog boxes and pop-up widows
Creating forms that respond to user input
Displaying date and time

Control Flow and Loops

In JavaScript, control flow refers to code being executed from top to bottom line by line unless it encounters a statement that alters the programs control flow. These statements can be Loops, Functions or Conditionals.

Loops are iterative (repetitive) statements that repeat continuously until the condition is false or there is no more data to read. It can be compared to reading a book like the adventure style goosebumps book. You read up to a certain point, choose an action, flick to the page it tells you to go to, read on and you either continue the adventure or die.

Describe what the DOM is and an example of how you might interact with it

The Document Object Model or DOM is an object of representation of your whole document (HTML and CSS). The browser that you use to view a web page creates a representation of the code so that programs can access and manipulate the the document structure, style and content. The DOM is the glue that holds HTML, CSS together. Everything that makes up the DOM is categorized into nodes and objects.

You can use your development tools to interact with the DOM and use a programming language to manipulate the content, structure and style of a website. For example you can select HTML tags like Classes, Ids (#) and Elements using the document.getElementById() and the querySelector()

Explain the difference between accessing data from arrays and objects

An Array stores data in an indexed ordered collection and can be accessed through using a numerical index. An array starts at 0 and is a single variable that can be added to an object as an attribute.

For example:
0 (Rafael) 1 (Sarah) 2 (James) 3 (Connor)

An object is a data structure in JavaScript which is used to represent a ‘thing’. The ‘thing’ can be defined as an object like a person, car, or item with key information. Objects contain information inside their attributes. An object can hold multiple attribute with values. For example:

Object: Rafael
Age: 28
Ethnicity: Kiwi
Job: HVAC Mechanic
Favourite Food: Anything with cheese

Object attributes can be stored as numbers, booleans, arrays, strings and more. Objects can be accessed by calling an objects name and it will return all the data attributes linked to the object. You can call specific attributes by calling ‘object.attribute’.

Explain what functions are and why they are helpful.

A function is a series of instructions used to perform a task. The function uses a list of instructions that it will use to complete a task or calculate a value, (output from its input).

A good example of a function would be when when you go to the grocery store and end up at the checkout. All the values of the food you want to purchase enter a function and the output is the total value you need to pay.
